激活函数是神经网络引入非线性表达能力的核心组件。本文梳理了Sigmoid、Tanh、ReLU 家族、Swish、GELU 等激活函数的原理、特性与选型策略。

<!--more-->

## 一、引言：为什么需要激活函数

### 1.1 激活函数的定义

在神经网络中，输入经过加权求和后，需要再经过一个函数的作用——这个函数就是**激活函数（Activation Function）**。

{{< image src="/images/activate-function-0.png" alt="神经元中的激活函数" caption="图：神经元中的激活函数位置示意，输入加权求和后经激活函数输出" >}}

### 1.2 激活函数的目的

首先需要明确：如果在神经网络中不引入激活函数，那么在该网络中，每一层的输出都是上一层输入的线性函数，无论最终神经网络有多少层，输出都是输入的线性组合。这种纯线性堆叠只能应用于线性分类问题，例如最原始的感知机（Perceptron）。没有激活函数的每层本质上都相当于矩阵相乘——叠加若干层之后，无非还是另一个矩阵相乘罢了。

关于神经网络中的激活函数的作用，通常都是这样解释：不使用激活函数的话，神经网络的每层都只是做线性变换，多层输入叠加后也还是线性变换。因为线性模型的表达能力通常不够，所以这时候就体现了激活函数的作用了，激活函数可以引入非线性因素。

使用非线性激活函数是为了增加神经网络模型的非线性因素，以便使网络更加强大，增加它的能力，使它可以学习复杂的事物、复杂的表单数据，以及表示输入输出之间非线性的复杂的任意函数映射。输出层有时可能使用线性激活函数，但在隐含层一般都使用非线性激活函数。

更进一步的，激活函数在神经网络中的应用，除了引入非线性表达能力，其在**提高模型鲁棒性**、**缓解梯度消失问题**、**将特征输入映射到新的特征空间**以及**加速模型收敛**等方面都有不同程度的改善作用。

正如绝大多数神经网络借助某种形式的梯度下降进行优化，激活函数需要是可微分的（或者至少是几乎完全可微分的）。此外，复杂的激活函数也可能产生一些梯度消失或梯度爆炸的问题。

## 二、激活函数的分类与通用特性

{{< image src="/images/activate-function-1.png" alt="激活函数分类" caption="图：激活函数分类概览，按饱和/非饱和等维度划分" >}}

从宏观上，激活函数可按是否"饱和"分为两大类：**饱和激活函数**（如 Sigmoid、Tanh）在输入绝对值较大时梯度趋近于零；**非饱和激活函数**（如 ReLU 及其变体）则在某些区间保持非零梯度，更利于深层网络的训练。更细致的分类可以按函数形态分为：恒等函数（Identity）、阶跃函数（Step）、S 形函数（Sigmoidal）和斜坡函数（Ramp）四大类。其中 Sigmoid 和 Tanh 同属 S 形函数家族，区别在于值域是否关于原点对称（Binary 型输出 0/1，Bipolar 型输出 -1/1）。

在选择激活函数时，以下通用特性是权衡的依据：

- **非线性（Nonlinearity）**：两层神经网络配合非线性激活即可满足通用近似定理。恒等激活函数不满足此特性，等价于单层网络。
- **值域（Range）**：有限值域使基于梯度的训练更稳定（对权重的更新影响有限）；无限值域则训练通常更有效（权重更新更明显），但需配合较小的学习率。
- **连续可微（Continuously differentiable）**：基于梯度的优化方法要求激活函数可微。ReLU 在 0 处不可微不是连续可微的，但实践中取单侧导数即可工作，它仍能使用梯度优化。而二值阶跃函数在 0 处不可微且其他地方导数为零，因此梯度下降完全无法适用于它。
- **单调性（Monotonicity）**：单调激活函数保证单层模型的误差曲面为凸面，即误差函数是凸函数，求得的最小值必为全局最小值，不致陷入局部最优。
- **一阶导单调（Smooth functions with a monotonic derivative）**：若导函数本身也单调，通常网络表现更好，训练过程更稳定。
- **原点近似恒等（Approximates identity near origin）**：若激活函数满足 `f(0) ≈ 0` 且 `f'(0) ≈ 1`，在随机初始化较小权重时学习更高效。不具备此特性的激活函数（如 Sigmoid），初始化权重时必须特别小心。

## 三、经典激活函数

### 3.1 Sigmoid

Sigmoid 是早期神经网络中最常用的激活函数，它将任意实数映射到 `(0, 1)` 区间，天然适合表示概率。

```text
sigmoid(x) = 1 / (1 + exp(-x))
导数 = sigmoid(x) · (1 - sigmoid(x))
```

{{< image src="/images/activate-function-2.jpeg" alt="Sigmoid 函数及其导数" caption="图：Sigmoid 函数图像及其导数曲线，两端饱和导致梯度消失" >}}

**定位**：将输出映射为概率值。

**核心场景**：
- 二分类任务的输出层：`(0, 1)` 的值域天然契合"是/否"的概率表示。
- LSTM/GRU 的门控单元：0 到 1 的输出值决定了信息流动的比例。

**主要问题**：
- **梯度消失**：当输入绝对值过大或过小时，导数趋近于 0。深层网络反向传播时梯度逐层衰减，靠近输入层的参数几乎无法更新。
- **输出非零中心（non-zero-centered）**：输出恒为正，导致后续层的梯度更新出现"Z 字形"抖动，影响收敛速度。

### 3.2 Tanh（双曲正切）

Tanh 可以看作 Sigmoid 的"零中心化"改进版，将输入压缩到 `(-1, 1)` 区间。

```text
tanh(x) = (exp(x) - exp(-x)) / (exp(x) + exp(-x))
导数 = 1 - tanh²(x)
```

{{< image src="/images/activate-function-3.png" alt="Tanh 函数" caption="图：Tanh 函数图像，关于原点对称，输出范围为 (-1, 1)" >}}

**定位**：解决了 Sigmoid 的非零中心问题，均值为 0，通常比 Sigmoid 收敛更快。

**核心场景**：在 ReLU 流行之前，常用于 RNN 或多层感知机的隐藏层。

**主要问题**：梯度饱和问题依然存在——输入过大或过小时导数同样趋近于 0，梯度消失风险未根本消除。

### 3.3 梯度消失与梯度爆炸详解

梯度消失（Vanishing Gradient）和梯度爆炸（Exploding Gradient）是深度神经网络训练中最致命的两类数值问题，它们都源于反向传播中链式法则的连乘效应。

**反向传播的连乘本质**

考虑一个 `L` 层的网络。根据链式法则，损失 `J` 关于第 `l` 层参数 `W⁽ˡ⁾` 的梯度是 `L-l` 个矩阵的乘积：

```text
∂J/∂W⁽ˡ⁾ = (∂J/∂o) · (M⁽ᴸ⁾ · M⁽ᴸ⁻¹⁾ · ... · M⁽ˡ⁺¹⁾) · (∂h⁽ˡ⁾/∂W⁽ˡ⁾)
```

其中每个 `M⁽ᵏ⁾` 包含了该层线性变换和激活函数导数的信息。当 `L-l` 很大时（即深层网络的早期层），连乘的结果极易出现数值问题。

**梯度消失**

当连乘的每个因子都小于 1 时，乘积随着层数增加呈指数级衰减：

- Sigmoid 的导数最大值为 **0.25**，且当输入绝对值增大时迅速趋近于 0。一个 10 层网络的第一层，反向传播的梯度中已经连乘了 9 个 ≤0.25 的因子，梯度量级约为 `(0.25)⁹ ≈ 3.8×10⁻⁶`——几乎为零。
- Tanh 的导数最大值虽然为 1，但只要输入稍微偏离 0，导数就会急剧减小，在深层网络中同样面临连乘衰减。

**后果**：靠近输入层的梯度几乎为零 → 这些层的权重几乎不更新 → 网络的前几层实际上没有学到任何东西，等于"白堆了"层数。模型表现出"深层不如浅层"的反常现象——不是因为深度有害，而是因为浅层压根没被训练。

**梯度爆炸**

相反，当连乘因子都大于 1 时，梯度呈指数级增长。例如连续乘 100 个方差为 1 的随机矩阵，结果矩阵的元素量级可达 10²⁴ 级别。如果初始化不当导致权重过大，配合某些激活函数后梯度会飞速膨胀。

**后果**：参数更新步长巨大 → 权重在一次更新后跳到极端值 → 损失函数变成 `NaN` 或 `Inf` → 训练崩溃。

**为什么 ReLU 能缓解梯度消失**

回顾 ReLU 的导数：正区间恒为 **1**，负区间为 0。在正区间，连乘因子等于 1 而非小于 1，梯度不会衰减；负区间虽然梯度为 0（神经元死亡），但至少不会指数衰减。这也是为什么 ReLU 系列能支持上百层的网络训练——只要信号始终在正区间传播，梯度就能几乎无损地反传到底层。

**但注意**：ReLU 仍然可能梯度爆炸（如果权重初始化过大），且负区间的 0 梯度导致了 Dying ReLU 问题。这正引出了下一节的改进方案。

## 四、ReLU 家族：现代神经网络的基石

### 4.1 ReLU（修正线性单元）

ReLU 的诞生是深度学习的重要里程碑，以惊人的简洁高效迅速成为绝大多数现代网络的默认激活函数。

```text
ReLU(x) = max(0, x)
导数：x > 0 时为 1，x < 0 时为 0，x = 0 处取 0
```

{{< image src="/images/activate-function-4.png" alt="ReLU 函数及其导数" caption="图：ReLU 函数图像及其导数，正区间梯度恒为 1，有效缓解梯度消失" >}}

**定位**：现代深度学习的标准激活函数，隐藏层的默认首选。

**核心优势**：
- **极大缓解梯度消失**：正区间导数恒为 1，梯度能顺畅传播。
- **计算高效**：仅需一次 `max` 比较，无指数运算，远快于 Sigmoid/Tanh。
- **稀疏性**：负区间输出为 0，使网络具备一定的稀疏性，客观上起到正则化效果。

**主要风险——神经元死亡（Dying ReLU）**：如果某个神经元在训练中始终收到负输入，其梯度永远为 0，参数永远无法再更新。这通常由过大的学习率或不当的初始化引发。

### 4.2 Leaky ReLU 与 PReLU

为解决 Dying ReLU 问题，在负区间引入一个微小的非零斜率，保证梯度不会完全中断。

```text
Leaky ReLU(x) = max(αx, x)，其中 α 为小常数（如 0.01）
PReLU(x) = max(αx, x)，其中 α 为可学习参数
```

**定位**：ReLU 的直接修复方案。

{{< image src="/images/activate-function-5.png" alt="Leaky ReLU 与 PReLU" caption="图：Leaky ReLU / PReLU 函数图像，负区间引入微小斜率避免神经元死亡" >}}

**核心场景**：当怀疑模型存在大量死亡神经元，或希望进一步压榨性能时，可作为 ReLU 的替代品。PReLU 的 `α` 通过反向传播学习，灵活性更高，但也增加了过拟合风险。

## 五、前沿激活函数：追求更优的平滑非线性

### 5.1 Swish / SiLU

Swish（在 PyTorch 中命名为 SiLU）是一种"自门控"（self-gated）激活函数，在多项计算机视觉任务上稳定优于 ReLU。

```text
Swish(x) = x · sigmoid(x) = x / (1 + exp(-x))
```

{{< image src="/images/activate-function-6.png" alt="Swish 函数" caption="图：Swish/SiLU 函数图像，平滑非单调，在负区间有微小凹陷" >}}

**定位**：ReLU 的强力平滑替代品，尤其适用于计算机视觉。

**核心场景**：EfficientNet、YOLOv7+ 等现代 CNN 架构。

**为何有效**：
- **平滑非单调**：曲线平滑且负区有一个小"凹陷"，增强表达能力。
- **自门控**：`sigmoid(x)` 部分充当一个软"门"，决定输入 `x` 有多大比例能通过。
- **无上界、有下界**：避免梯度饱和，负区抑制效果兼具正则化作用。

### 5.2 GELU（高斯误差线性单元）

GELU 是大模型时代的标配，将非线性与随机正则化思想巧妙结合。

```text
GELU(x) = x · Φ(x)，其中 Φ(x) 是标准高斯分布的累积分布函数
近似计算：GELU(x) ≈ 0.5x · (1 + tanh(√(2/π) · (x + 0.044715x³)))
```

{{< image src="/images/activate-function-7.png" alt="GELU 函数" caption="图：GELU 函数图像，可看作 Swish 的平滑近似，与随机 dropout 有深刻联系" >}}

**定位**：Transformer 架构的标配激活函数。

**核心场景**：BERT、GPT 系列等所有主流 Transformer 模型的前馈网络（FFN）层。

**为何有效**：GELU 可以理解为——神经元的输出是其输入 `x` 与一个伯努利随机掩码的乘积，而掩码的概率由 `x` 本身决定（输入越大，被保留的概率越高）。这种**依赖于输入的自适应随机性**与 Dropout 有深刻的数学联系，被证实极度适合深层 Transformer 模型，有助于提升训练稳定性和最终性能。形象地说，GELU 在"确定性非线性"和"随机正则化"之间找到了最佳平衡点。

## 六、输出层激活函数：任务的"最后一公里"

### 6.1 Softmax

Softmax 不常用于隐藏层，它专为多分类问题的输出层设计，将网络的原始输出（logits）转换为概率分布。

```text
Softmax(z_i) = exp(z_i) / Σ_j exp(z_j)
```

**定位**：将 logits 向量转化为多类别上的概率分布。

**核心场景**：多分类模型的输出层。确保所有输出类别的概率之和为 1，与"从多个互斥选项中选择一个"的目标完美契合。二分类场景下，Sigmoid 是其等价替代。

> 关于 Softmax 与交叉熵损失的详细原理及从零实现，可参见本站的 [softmax 回归与交叉熵损失](../softmax%20and%20cross-entropy%20loss/) 笔记。

## 七、总结与工程选型指南

### 7.1 核心特性对比

| 激活函数 | 公式 | 值域 | 优点 | 主要问题 |
|---------|------|------|------|---------|
| Sigmoid | `1/(1+e⁻ˣ)` | (0, 1) | 概率解释、门控 | 梯度消失、非零中心 |
| Tanh | `(eˣ-e⁻ˣ)/(eˣ+e⁻ˣ)` | (-1, 1) | 零中心、比 Sigmoid 更快 | 仍梯度饱和 |
| ReLU | `max(0, x)` | [0, ∞) | 无饱和、计算快、稀疏 | 神经元死亡 |
| Leaky ReLU | `max(αx, x)` | (-∞, ∞) | 修复死亡神经元 | α 为超参需调试 |
| Swish/SiLU | `x·sigmoid(x)` | (-0.28, ∞) | 平滑非单调、自门控 | 计算量略高于 ReLU |
| GELU | `x·Φ(x)` | (-0.17, ∞) | 自适应随机正则化 | 计算量高于 Swish |
| Softmax | `eᶻⁱ/Σeᶻʲ` | (0, 1)ⁿ | 概率分布输出 | 仅适用输出层 |


### 7.2 选型黄金法则

在实际工程中，可以遵循以下简洁的决策路径：

1. **通用起点**：隐藏层无特殊理由时，直接选 **ReLU**。在绝大多数场景下稳健高效。
2. **CNN 架构优化**：构建现代卷积网络时，强烈建议用 **Swish/SiLU** 替代 ReLU，通常能带来免费的性能提升。
3. **Transformer 架构**：无需犹豫，直接使用 **GELU**，这是 BERT/GPT 等模型的工业标准。
4. **输出层**：二分类用 **Sigmoid**，多分类用 **Softmax**。
5. **问题排查**：怀疑 ReLU 导致大量神经元死亡时，尝试 **Leaky ReLU** 诊断或修复。

### 一条贯穿全文的主线

激活函数的发展脉络可以概括为三个阶段：(1) **Sigmoid/Tanh 时代**——有界、平滑但致命地易饱和；(2) **ReLU 时代**——放弃部分平滑性换来极致的梯度流和计算效率，成为深度学习的默认选择；(3) **后 ReLU 时代（Swish/GELU）**——在保持梯度健康的前提下，通过平滑性和自适应机制重新找回有界激活函数的优点。理解这条主线，就不会把激活函数当成"随便选一个"的超参数，而是能根据架构和任务做出有依据的选择。
