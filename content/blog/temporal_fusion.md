时序融合是将多个连续时间步的视觉特征进行整合，挖掘帧与帧之间的时序依赖关系（运动趋势、状态变化、上下文关联），最终输出包含时空信息的融合特征。

<!--more-->

## 一、基础概念与问题背景

### 1.1 什么是时序融合

时序融合是视频理解、时序视觉任务（如自动驾驶多帧感知、动作识别、轨迹预测）中的核心步骤，目标是将多个连续时间步的视觉特征进行整合，挖掘帧与帧之间的时序依赖关系（运动趋势、状态变化、上下文关联），最终输出包含时空信息的融合特征，供下游分类、检测、分割等任务使用。

### 1.2 ViT 架构下的时序融合输入输出形态

在基于ViT的多帧任务中，单帧图像会被拆分为N个patch，每个patch映射为D维特征向量，单帧特征形状为 `[B, N, D]`。当输入T个历史帧时，原始特征张量为 `[B, T, N, D]`，各维度定义如下：

- **B**：batch size，批量大小
- **T**：时间帧数（常见场景 T=4）
- **N**：单帧的空间patch数量（如224×224图像、patch_size=16时，N=196）
- **D**：patch特征维度（如ViT-Base中 D=768）

时序融合的核心操作对象是**时间维度T**，最终目标通常是将T维的时序信息交互、压缩后，输出 `[B, N, D]` 的单帧融合特征，或输出增强后的 `[B, T, N, D]` 时序特征。

## 二、时序融合的层级划分

按照融合发生在特征流水线的阶段，时序融合可分为三大类，对应不同的工程代价与信息保留度，也是学术论文中最基础的分类范式。

### 2.1 像素级融合（早期融合）

在原始输入层面完成时序信息合并，再送入主干网络。常见操作包括：多帧通道堆叠（将T帧RGB图像拼接为3T通道输入）、帧差计算、光流图拼接等。

- **优势**：保留最完整的底层像素级时序信息，理论上信息损失最小。
- **劣势**：计算量随帧数线性增长，破坏单帧网络的输入结构，无法直接复用图像预训练权重；且帧间运动错位会导致特征对齐困难。
- **适用场景**：帧数极少、算力充足的小分辨率任务，或光流等专用运动特征输入。

### 2.2 特征级融合（中期融合）

单帧图像先经过主干网络提取高维特征，再在特征空间完成时序交互与聚合。4帧ViT方案就属于典型的特征级融合。

- **优势**：平衡精度与计算开销，主干网络可完整复用图像预训练权重，时序模块可灵活插拔，是当前工业界与学术界的绝对主流。
- **劣势**：底层像素的细粒度时序动态会在特征提取过程中部分丢失。
- **工程地位**：Temporal Transformer、1D Conv、LSTM、池化等所有方案，本质都是特征级融合的不同实现。

### 2.3 决策级融合（晚期融合）

每一帧独立完成下游任务推理（分类、检测等），再对输出的结果进行时序聚合，如投票、加权平均、跟踪关联等。

- **优势**：解耦性最强，单帧模型完全无需修改，部署灵活，容错率高（单帧失效不影响整体）。
- **劣势**：丢失了特征层面的时序上下文，无法利用时序信息提升单帧本身的特征质量，精度上限最低。
- **适用场景**：多模型集成、低算力边缘端、对系统稳定性要求极高的工业系统。

> 针对4帧ViT的场景：**特征级融合是性价比最高的选择**，既可以复用ViT预训练权重，又能通过时序交互提升特征质量，工程改造成本可控。

## 三、时序融合技术的工程演变脉络

时序融合的技术演进并非线性的"学术升级"，而是始终围绕**算力约束、基础架构迭代、工业落地需求**三大核心驱动力螺旋发展。从早期手工特征时代到如今的ViT大一统架构，每一代方案的出现与淘汰，本质都是在特定硬件与任务条件下，对"精度-速度-部署成本"三角的重新平衡。理解这段演变历史，有助于理解为什么当前的四种主流方案（Temporal Transformer、池化、1D Conv、LSTM）会成为今天的标准选项。

### 3.1 萌芽期：传统手工特征时代（2010年以前）

深度学习尚未普及，视觉特征依赖人工设计（HOG、SIFT、Haar特征等），算力仅能支撑低维特征计算，时序融合的核心目标是"用最少的计算量捕捉运动信息"。

主流融合思路：
1. **像素级早期融合**：帧差法、背景建模法，直接对像素值做时序差分，提取运动区域。计算量极低，但对光照、遮挡鲁棒性差，仅适用于简单监控场景。
2. **特征级加权融合**：提取单帧手工特征后，对连续帧特征做滑动窗口加权、光流特征拼接，通过人工设计的权重体现时序相关性。
3. **序列概率建模**：用隐马尔可夫模型（HMM）、条件随机场（CRF）对时序特征序列建模，建模状态转移概率，多用于简单动作识别。

这一阶段奠定了时序融合的两大基础范式：**特征聚合**与**序列建模**，后续所有深度学习方案本质都是这两种思路的高维延伸。

### 3.2 迭代期：CNN架构主导的工程化探索（2012-2020）

AlexNet开启深度学习时代后，2D CNN在图像任务上快速成熟，视频时序融合进入"基于CNN backbone做时序扩展"的十年，期间经历了三次明显的工程路线摇摆。

**早期：2D CNN + 晚期池化（最简落地范式）**
- 复用成熟的图像2D CNN模型，逐帧提取特征，在网络输出端通过均值/最大值池化完成时序融合。
- 改造成本几乎为零，可完美复用图像预训练权重，推理速度与单帧CNN几乎无差异。
- 完全丢失时序顺序与运动信息，仅能作为基线方案。对应后文的 **Mean/Max Pooling** 方案。

**中期：3D CNN的技术爆发与落地困境**
- 将2D卷积扩展为3D卷积，卷积核同时在空间与时间维度滑动，端到端同时提取时空特征（C3D、I3D、P3D、3D ResNet等）。
- 学术上首次证明了端到端时空联合建模的精度上限，但参数量与计算量随帧数线性暴涨，显存占用极高，端侧硬件兼容性差。除了少数云端离线场景，工业界几乎未大规模落地。

**后期：2D Backbone + 轻量时序模块（工业落地主流）**
- 工业界对推理速度、显存占用、部署兼容性的硬性要求，倒逼行业放弃3D CNN，回归"2D主干+轻量时序补丁"的折中路线。
- 代表工程方案：TSN（时序分段网络）、TSM（时间移位模块）、SlowFast双支路网络，以及1D Conv、LSTM的插入式融合。
- 以成熟2D CNN为绝对主体，时序模块仅做"增量补充"，算力开销控制在整体的10%以内。对应后文的 **1D Conv**、**LSTM/GRU** 方案，至今仍是边缘端、嵌入式场景的主流选择。

### 3.3 成熟期：Transformer时代的时空解耦与统一架构（2020至今）

ViT在图像领域验证了纯注意力架构的潜力后，视频时序融合进入"时空注意力解耦"的新阶段，也是当前4帧ViT项目所处的技术阶段。

**早期探索：全时空自注意力的算力陷阱**
- 最初思路：直接将视频所有帧的所有patch拼接成一条长序列，做全局自注意力（如ViViT的全注意力变体）。
- 序列长度变为 `T×N`，自注意力复杂度为 `O((T×N)²·D)`，相比单帧ViT暴涨T²倍，算力开销完全不可接受，仅能在极短序列、小分辨率的学术实验中使用。

**工程落地：时空解耦的Temporal Transformer**
- 以TimeSformer为代表，将空间注意力与时间注意力完全拆分、交替执行：
  1. **空间注意力**：每一帧独立做patch间的自注意力，处理空间信息，复杂度 `O(T·N²·D)`；
  2. **时间注意力**：每个空间位置独立做帧间的自注意力，处理时序信息，复杂度 `O(N·T²·D)`。
- 对于短时序场景（如T=4），时序注意力的开销远小于空间注意力，整体算力增幅不足20%。时序模块与空间模块都是标准Transformer算子，可直接复用ViT的预训练权重、优化策略、部署工具链。

**进阶优化：面向感知任务的交叉注意力变体**
- 自动驾驶检测、跟踪等感知任务具有"当前帧为主、历史帧为辅助"的强先验。
- 仅以当前帧特征为Query，所有历史帧特征为Key和Value，做帧间交叉注意力，复杂度从 `O(T²)` 降为 `O(T)`。

### 3.4 新趋势：部署导向的专业化演进（2023至今）

1. **端侧轻量化方向**：通过时序知识蒸馏、稀疏关键帧融合、硬件算子融合等手段，将Transformer级别的时序能力压缩到端侧芯片。
2. **流式在线推理方向**：发展增量式时序融合——复用历史帧的计算缓存、隐藏状态，LSTM/GRU等循环结构凭借天然的流式特性重新获得工程关注。
3. **大模型统一方向**：视频大模型中将时序建模与多模态对齐融合，时序模块与文本、音频等模态的序列建模共用统一的Transformer架构。

### 3.5 演变的核心驱动规律

1. **算力约束是底层逻辑**：从手工特征到3D CNN再到时空解耦，所有技术路线都在"精度上限"与"算力成本"之间动态平衡。
2. **基础架构决定融合范式**：CNN时代是"空间为主、时序为辅"的补丁式融合；Transformer时代是时空平等的解耦式融合。
3. **落地需求倒逼方案迭代**：工业界对推理速度、显存、部署兼容性的要求，持续推动学术方案向轻量化、工程友好型演进。
4. **预训练复用性成为核心选型指标**：能否复用成熟的图像预训练权重，直接决定了方案的冷启动成本与数据效率。

## 四、四种时序融合方案详解

### 4.1 Temporal Transformer（时序自注意力）

#### 核心原理

借鉴Transformer的自注意力机制，**沿时间维度计算自注意力，每个空间位置独立进行时序交互**。本质是将"空间位置"视为批量，"时间帧"视为序列长度，让每个空间位置的特征在时间维度上做注意力加权，自动学习不同时间帧的贡献权重。

这种设计也被称为"时空解耦注意力"，是TimeSformer、Video Swin Transformer等经典视频Transformer中时序建模的核心思路。

#### 计算流程与实现逻辑

1. **维度变换**：将输入特征 `[B, T, N, D]` 重排，把空间维度N与批量维度B合并，得到形状为 `[B*N, T, D]` 的张量。此时每一条序列对应一个空间位置在T个时间步的特征。
2. **自注意力计算**：对上述张量执行标准Transformer Encoder操作（多头自注意力 + 前馈网络FFN）。Q、K、V均由T维时序特征线性投影得到，注意力计算在T个时间步之间进行，且每个空间位置的计算完全独立。
3. **维度恢复**：将输出的 `[B*N, T, D]` 张量reshape回 `[B, T, N, D]`，也可进一步在时间维聚合得到 `[B, N, D]`。

**伪代码（PyTorch风格）**

```python
import torch
import torch.nn as nn
from einops import rearrange

class TemporalTransformer(nn.Module):
    def __init__(self, dim, num_heads=8, depth=1):
        super().__init__()
        # 单层Transformer Encoder，处理时间维度
        encoder_layer = nn.TransformerEncoderLayer(
            d_model=dim,
            nhead=num_heads,
            dim_feedforward=dim*4,
            batch_first=True
        )
        self.transformer = nn.TransformerEncoder(encoder_layer, num_layers=depth)

    def forward(self, x):
        # x: [B, T, N, D]
        B, T, N, D = x.shape
        # 重排：空间维度转为batch，时间维度作为序列长度
        x = rearrange(x, 'b t n d -> (b n) t d')
        # 时序自注意力交互
        x = self.transformer(x)  # 输出: [(B*N), T, D]
        # 恢复原始维度
        x = rearrange(x, '(b n) t d -> b t n d', b=B, n=N)
        # 可选：压缩时间维得到单帧融合特征
        # x = x.mean(dim=1)  # [B, N, D]
        return x
```

**计算复杂度**：注意力部分复杂度为 `O(B*N * T² * D)`。对于T=4的场景，T²=16，计算开销极低，远小于空间注意力的 `O(B*T*N²*D)`（N通常为数百），几乎不会增加推理延迟。

#### 优势与局限

**优势**
1. 建模能力最强：自注意力可以动态学习帧间依赖权重，精准捕捉不同帧的相关性，同时支持局部与长程时序关系建模。
2. 架构原生统一：与ViT核心算子完全一致，训练调参、部署优化的经验可以直接复用，无需引入新算子。
3. 可解释性强：可通过可视化注意力权重，直观看到每个空间位置上各历史帧的贡献程度，便于分析调优。
4. 高度并行：所有时间步的计算可并行执行，训练效率高。

**局限**
1. 理论计算量随T²增长，帧数T很大时（如T=32）开销显著上升；但T=4时无性能压力。
2. 数据依赖性强：小数据集上注意力权重可能训练不充分，效果未必优于简单方案。
3. 参数量略高于轻量方案：单层Transformer包含QKV投影、两层FFN，参数量约为 4×D²。

#### 适用场景

- 对精度要求高、帧数适中（T≤8）的离线/准实时任务
- 基于ViT架构的视觉任务，希望保持架构统一性
- 需要分析帧间贡献权重、可解释性要求高的研究场景

### 4.2 Mean / Max Pooling（均值/最大值池化）

#### 核心原理

无参数的时序融合方案，直接在时间维度T上执行均值池化或最大值池化，将T个时间步的特征压缩为1个特征向量：

- 均值池化：所有帧的特征取平均，默认所有帧贡献相等。
- 最大值池化：每个特征维度取T帧中的最大值，保留最显著的特征响应。

#### 实现方式

```python
# 输入 x: [B, T, N, D]
x_mean = torch.mean(x, dim=1)  # 均值融合，输出 [B, N, D]
x_max, _ = torch.max(x, dim=1) # 最大值融合，输出 [B, N, D]
```

#### 优势与局限

**优势**
1. 实现极简：零参数量、零额外计算量，推理速度几乎无损耗。
2. 无训练成本：无需额外参数训练，可直接使用。
3. 稳定性强：不会引入额外过拟合风险。

**局限**
1. **完全丢失时序顺序信息**：均值池化打乱帧顺序后结果完全不变，无法建模运动、变化等时序动态。
2. 权重固定：所有帧同等重要，无法区分当前帧与历史帧的优先级，也无法自适应调整不同场景下的帧贡献。
3. 信息损失大：最大值池化只保留峰值特征，丢失大量时序上下文；均值池化会平滑掉关键时序差异。

#### 适用场景

- 基线（Baseline）方案，用于验证时序融合的收益上限
- 帧数极少、场景几乎无变化的低要求任务
- 算力极度受限的边缘部署场景

### 4.3 1D 卷积（1D Conv）

#### 核心原理

使用一维卷积核对时间维度T进行卷积操作，通过可学习的卷积核提取**局部时序特征**。核心思想是：相邻帧之间的关联最紧密，用局部滑动窗口建模时序变化。卷积核大小k决定了时序感受野，k越大能捕捉的时序范围越广。

#### 实现方式

通常将特征维度D作为通道维度，时间维度T作为序列长度，执行1D卷积。若需直接融合为单帧，可使用kernel_size=T的全局1D卷积，等价于一个作用于时间维度的全连接层。

```python
class TemporalConv1D(nn.Module):
    def __init__(self, dim, kernel_size=3):
        super().__init__()
        self.conv = nn.Conv1d(
            in_channels=dim,
            out_channels=dim,
            kernel_size=kernel_size,
            padding=kernel_size // 2,  # 保持时间长度不变
            groups=dim  # 深度可分离卷积，进一步轻量化
        )

    def forward(self, x):
        # x: [B, T, N, D]
        B, T, N, D = x.shape
        # 重排：D作为通道，T作为序列长度
        x = rearrange(x, 'b t n d -> (b n) d t')
        x = self.conv(x)  # 输出: [(B*N), D, T]
        # 恢复原始维度
        x = rearrange(x, '(b n) d t -> b t n d', b=B, n=N)
        return x
```

#### 优势与局限

**优势**
1. 轻量高效：参数量和计算量都很小，计算复杂度为 `O(B*N*T*D*k)`，k远小于T时开销极低。
2. 能捕捉局部时序动态：相比池化，有能力学习帧间的局部变化趋势（如运动梯度），保留时序顺序信息。
3. 部署友好：1D卷积是硬件高度优化的算子，推理速度快，延迟稳定。

**局限**
1. 感受野有限：只能捕捉卷积核范围内的局部依赖，长程时序建模能力弱。
2. 权重静态：卷积核训练完成后权重固定，无法根据输入内容动态调整帧间权重，自适应能力弱于注意力。
3. 需多层堆叠才能扩大感受野，会增加网络深度和训练难度。

#### 适用场景

- 轻量级模型、边缘部署场景
- 时序关联性集中在相邻帧的短时序任务
- 作为复杂时序融合模块的轻量化替代方案

### 4.4 LSTM / GRU（循环神经网络）

#### 核心原理

基于循环神经网络的门控机制，逐个时间步处理特征，通过输入门、遗忘门、输出门（LSTM）控制时序信息的传递与遗忘，建模序列的时序依赖。

与时序Transformer类似，每个空间位置独立处理，将每个空间位置的T帧特征作为一条序列输入LSTM/GRU，通常取最后一个时间步的隐藏状态作为融合后的特征。

#### 实现方式

```python
class TemporalLSTM(nn.Module):
    def __init__(self, dim, hidden_dim=None):
        super().__init__()
        hidden_dim = hidden_dim or dim
        self.lstm = nn.LSTM(
            input_size=dim,
            hidden_size=hidden_dim,
            batch_first=True,
            bidirectional=False  # 在线推理用单向，离线任务可用双向
        )

    def forward(self, x):
        # x: [B, T, N, D]
        B, T, N, D = x.shape
        x = rearrange(x, 'b t n d -> (b n) t d')
        # LSTM前向传播，取最后时刻的隐藏状态
        output, (hn, cn) = self.lstm(x)
        x = hn.squeeze(0)  # [(B*N), D]
        # 恢复空间维度
        x = rearrange(x, '(b n) d -> b n d', b=B, n=N)
        return x
```

#### 优势与局限

**优势**
1. **在线推理友好**：天然支持串行逐帧输入，每来一帧更新一次隐藏状态，无需缓存所有历史帧，内存占用稳定，适合实时流式系统。
2. 门控筛选：能一定程度上筛选重要时序信息、遗忘无关信息，建模能力优于池化和简单1D Conv。
3. 参数量可控：单层LSTM参数量约为 4×D×hidden_size，与单层Transformer相当或略小。

**局限**
1. 并行性差：训练时必须按时间步串行计算，无法完全并行，训练速度慢于注意力和卷积。
2. 长序列建模弱：存在梯度消失问题，长时序下性能不如Transformer；但T=4的短序列下差距不明显。
3. 架构不统一：与ViT的Transformer架构差异大，调参逻辑不同，部署时需额外适配循环算子。
4. 单向建模：标准单向LSTM只能利用历史帧信息，无法双向建模。

#### 适用场景

- 实时在线推理、流式输入的部署场景
- 对延迟敏感、需要逐帧输出的系统
- 短时序、低算力的嵌入式场景

### 5.2 针对「4帧历史帧ViT」场景的选型建议

短时序视觉感知任务（如自动驾驶历史帧融合），结合各方案特性，推荐优先级如下：

**1. 首选主方案：Temporal Transformer**
- T=4时计算开销完全可以忽略，不会增加推理负担；
- 与ViT原生架构完全统一，训练时可直接复用ViT的优化器、正则化策略，调参成本低；
- 动态注意力权重能自适应学习当前帧与历史帧的贡献，精度上限最高；
- 可拓展性强，后续增加帧数、加入交叉注意力优化都很方便。

**2. 基线对照方案：Mean Pooling + 1D Conv**
- 用Mean Pooling作为最低基线，验证"时序融合"本身是否带来任务收益；
- 用1D Conv作为轻量基线，验证"可学习时序融合"的收益边界，量化Transformer方案的提升幅度。

**3. 备选方案：LSTM/GRU**
- 仅当项目有明确的"在线流式推理"需求时考虑；如果是离线训练、批量推理，LSTM无明显优势。

## 六、时序Transformer深度解析

### 6.1 时序位置编码（Temporal Positional Encoding）

#### 为什么必须有时序位置编码

自注意力机制本身具有**排列不变性**——如果不加入位置信息，打乱帧的顺序，注意力计算结果完全不变，等同于丢失了时序顺序信息，建模能力会退化为加权池化。因此时序位置编码是 Temporal Transformer 的必备组件，作用是向特征中注入"第几帧"的顺序信息。

#### 主流实现方案

**1. 可学习时序位置编码**
为每个时间步分配一个可学习的D维向量，直接与对应帧的特征相加。参数总量为 `T×D`，T=4时仅4个向量，参数量极小。
- 优点：实现简单，适配性强，在固定帧数的任务上效果最优。
- 缺点：泛化性差，训练时的帧数与推理不一致时性能下降明显。

**2. 正弦余弦固定位置编码**
沿用原始Transformer的正余弦位置编码公式，通过不同频率的三角函数生成位置向量，无需训练。
- 优点：零参数量，支持外推（推理帧数超过训练帧数也能工作），泛化性强。
- 缺点：短时序场景下，效果略逊于可学习编码。

**3. 相对时序位置编码**
不建模绝对帧序号，只建模帧与帧之间的相对距离（如当前帧与前1帧、前2帧的间隔），在注意力计算时注入相对位置偏置。
- 优点：对帧数变化鲁棒性最强，更符合"历史帧距离越远相关性越弱"的物理直觉。
- 缺点：实现稍复杂，需要修改注意力计算逻辑。

> T=4固定帧数场景，**优先使用可学习时序位置编码**，代码改动最小，收益最明显；如果后续需要扩展帧数或做流式推理，再切换为相对位置编码。

### 6.2 时空注意力的组合范式

在ViT架构中，空间注意力与时序注意力并非只能"先后执行"，工业界已经沉淀出多种成熟的组合模式，对应不同的精度-速度权衡。

#### 1. 晚期融合（独立串行）

- 逻辑：所有帧先完整走完所有空间Transformer层，得到单帧最终特征后，再统一做一次时序融合。
- 优点：代码改动最小，预训练权重100%复用，调试成本极低。
- 缺点：时空交互不充分，时序信息无法反向优化低层空间特征。
- 定位：快速验证方案、基线实验首选。

#### 2. 交替式融合（时空解耦）

- 逻辑：在Transformer backbone的每一层（或每隔几层），交替执行空间注意力与时序注意力，代表工作为TimeSformer：
  - 第1层：空间注意力 → FFN
  - 第2层：时序注意力 → FFN
  - 循环往复
- 优点：时空特征深度交互，精度上限最高，是当前学术与工业落地的主流范式。
- 缺点：需要修改backbone结构，预训练权重加载需要适配。

#### 3. 并行式融合

- 逻辑：同一层内，分别计算空间注意力与时序注意力，将两者的输出通过相加或拼接的方式融合，再送入FFN。
- 优点：时空信息同步交互，理论上信息流动更高效。
- 缺点：计算量与显存占用更高，工程收益不如交替式，实际使用较少。

> 推荐迭代路径：第一步用**晚期融合**快速跑通流程、验证时序收益；第二步将时序模块插入backbone最后2-3层，做**浅度交替融合**，在少量增加计算的前提下显著提升精度。

### 6.3 时序融合的插入位置

在ViT中，时序融合模块的插入位置有三种常见选择：

- **早期融合**：patch embedding之后、进入Transformer backbone之前融合，后续只做空间注意力。优点是计算量小，缺点是底层特征时序信息少。
- **中期融合**：backbone中间层交替执行空间注意力和时序注意力（如TimeSformer的时空交替结构）。优点是时空特征交互充分，效果最好。
- **晚期融合**：backbone输出最终特征后再做时序融合。优点是解耦性强，backbone可直接加载预训练权重，代码改动最小。

> 如果基于现有ViT快速迭代，建议先从晚期融合入手验证效果；后续追求精度再尝试中期交替融合。

### 6.4 Temporal Transformer 的轻量化变体

- **帧间交叉注意力**：只用当前帧作为Query，所有历史帧作为Key和Value，而非全量自注意力。计算量从O(T²)降为O(T)，更符合"历史帧辅助当前帧"的感知逻辑，精度损失很小。
- **减少头数与深度**：4帧短时序场景下，1层、4头的时序Transformer即可满足需求，可进一步压缩参数量。
- **加入时序位置编码**：为时间维度添加可学习的位置编码，补充帧的先后顺序信息，能小幅提升建模效果。

## 七、工程实践与调优

### 7.1 帧采样策略与时序对齐

#### 帧采样策略

并非所有时序任务都使用连续帧，不同的采样方式直接决定了时序感受野与建模难度：

- **连续采样**：选取时间上相邻的T帧（如t-3, t-2, t-1, t）。适合捕捉快速局部运动，也是自动驾驶历史帧融合的标准做法。
- **等间隔采样**：每隔固定帧数采一帧，在相同T下扩大时序覆盖范围。适合慢动作、长时序变化任务。
- **随机采样**：训练时在时间窗口内随机选取T帧，提升模型对不同帧间隔的鲁棒性。

#### 时序对齐问题

这是多帧视觉感知最容易被忽略、但影响极大的工程细节：由于相机运动、物体运动，不同帧的同一个空间patch对应的物理世界位置并不相同，直接做时序注意力会出现"错位匹配"，引入噪声。

- **隐式对齐**：不做额外处理，完全靠注意力机制自主学习对应关系。优点是零额外开销，缺点是数据量不足时对齐效果差。
- **显式对齐**：通过光流估计、相机位姿、目标检测框等先验信息，将历史帧特征通过仿射变换、可变形卷积等方式对齐到当前帧，再做融合。优点是精度高，缺点是引入额外计算量。

> 4帧短时序、运动幅度不大的场景，可先采用**隐式对齐**快速验证；如果后续发现运动物体区域精度提升不明显，再加入轻量级的显式对齐模块。

### 7.2 时序专属数据增强策略

单帧图像的翻转、裁剪、颜色抖动等增强可以直接复用，但时序任务有专属的增强手段，用于提升模型的时序鲁棒性、防止过拟合：

1. **帧丢弃增强（Frame Dropout）**：训练时随机丢弃1-2帧历史帧，用剩余帧完成融合，强制模型在帧缺失时仍能稳定输出。对于T=4的场景，可随机丢弃1帧，用3帧训练。

2. **时序打乱与顺序预测**：训练时以一定概率打乱帧顺序，同时增加一个辅助分类任务判断帧顺序是否正确。可以强制模型真正学习时序依赖，而非简单的特征平均。

3. **速度扰动**：通过调整帧间隔、重复帧、跳帧等方式，模拟不同运动速度的输入，提升模型对车速、物体运动速度变化的鲁棒性。

4. **时序一致增强**：对同一组视频帧施加相同的空间增强参数（如相同的裁剪区域、相同的翻转方向），保证时序空间一致性，避免引入伪运动。

### 7.3 训练技巧与常见坑

#### 预训练权重加载的冷启动问题

加入时序模块后，ViT预训练权重只能加载空间部分，时序模块的参数是随机初始化的。
- **正确做法**：训练分两阶段，第一阶段冻结backbone，只训练时序融合模块；收敛后再放开整体微调，且backbone使用远小于时序模块的学习率（如1:10的比例）。
- **常见错误**：直接整体大学习率训练，破坏预训练的空间特征，导致时序融合反而比单帧效果差。

#### 短时序的过拟合风险

T=4时，时序模块的参数量虽然不大，但如果数据集规模小，很容易出现"训练集涨点、验证集不涨"的过拟合现象。
- 应对：优先使用1层时序Transformer，不要盲目加深；加入帧丢弃增强、权重衰减；时序部分的Dropout率可略高于空间部分。

#### 帧数并非越多越好

历史帧并非越久越好，帧数超过阈值后，运动偏移增大、噪声增加，收益会边际递减，甚至出现负收益。
- 对于自动驾驶感知场景，通常3-5帧历史帧的性价比最高；T=4已经处于最优区间，无需盲目加到8帧以上。

#### 特征分布一致性

时序融合前后要保持特征尺度的一致性，建议在时序模块前后保留LayerNorm，避免数值分布突变破坏预训练特征的语义。

### 7.4 消融实验设计与评估维度

作为研究生实验，科学的消融实验是论文的核心支撑。针对时序融合任务，标准的实验设计框架如下：

#### 基线对照体系（必做）

| 实验组 | 实验目的 |
| :--- | :--- |
| 单帧基线 | 验证"无时序融合"的性能下限，量化所有时序方案的绝对收益 |
| Mean Pooling | 验证"引入时序信息但无学习能力"的基础收益 |
| 1D Conv | 验证"可学习局部时序建模"的收益边界 |
| Temporal Transformer | 验证"动态全局时序建模"的精度上限 |

#### 模块消融实验（深入分析）

- 时序位置编码：有/无的性能对比，验证位置信息的贡献
- 融合位置：早期/中期/晚期融合的对比，验证交互深度的影响
- 层数与头数：1层vs2层、4头vs8头的性能与效率对比
- 帧数敏感性：T=2/3/4/6的性能曲线，找到最优帧数性价比

#### 多维度评估指标

- **任务指标**：核心业务指标（如分类准确率、检测mAP、分割mIoU），衡量精度收益
- **效率指标**：参数量、计算量（FLOPs）、单帧推理延迟、峰值显存占用，衡量工程代价
- **鲁棒性指标**：不同帧率、不同运动速度、不同遮挡程度下的性能衰减，衡量方案稳定性

## 八、进阶研究方向

如果后续需要深入做学术创新，可关注几个前沿方向：

- **稀疏时序注意力**：只选取相关性高的关键帧做注意力，跳过冗余帧，进一步降低计算量。
- **动态时序融合**：根据输入场景的运动剧烈程度，自适应调整融合帧数、融合权重，实现精度与速度的动态平衡。
- **时序知识蒸馏**：用强时序教师模型监督单帧学生模型，让单帧模型获得接近多帧的性能，适配极致端侧部署。
- **多模态时序对齐**：将视觉时序与雷达、激光、文本等其他模态的时序信息做跨模态融合，是自动驾驶、多模态大模型的热点方向。
